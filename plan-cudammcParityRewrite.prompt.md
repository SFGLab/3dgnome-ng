# Plan: Rewrite Python to match cudaMMC

The audit identifies ~50 defects spanning every Python module. The fix is not local tweaks but a structural rewrite of the pipeline so it follows cudaMMC's **top-down hierarchical cascade**: segment-level heatmap MC → IB positioning via spline → per-IB arc MC on a **dense expected-distance matrix with -1 repulsion** → densify subanchors → smooth MC. The plan below sequences the rewrite so each phase is testable against `verify_algorithm.py` and a fresh cudaMMC run before the next phase lands.

## Further Considerations
1. **Single big rewrite vs. phased landings.** Recommend phased (above) — each phase is independently runnable and lets `verify_algorithm.py` catch regressions; a single PR touching all 11 source files will be impossible to review or bisect. Confirm preference.
2. **Root-cluster index reordering (§H6).** cudaMMC puts the root last; Python puts it at index 0 and all downstream code assumes that. Recommend keeping Python ordering and documenting the offset, unless you intend to read `.hcm` files produced by cudaMMC directly — in which case we must mirror the layout.
3. **Mirror upstream bugs verbatim?** The audit calls out two known cudaMMC oddities to copy: the `Settings.cpp:594-597` dist/angle swap (§E3) and the `findSplit` dead `exp_size` parameter (§H1 Branch B). Recommend mirroring exactly per the prime directive; flag both with `# cudaMMC: bug-preserved` comments for future-us.
4. **Inter-chromosomal arcs and density features.** Currently dropped silently (§I5, §G12). Recommend raising on unsupported INI flags rather than silently diverging — keeps "do X or fail loudly" guarantee.
